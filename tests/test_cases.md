# AI 编程与算法应用生成助手测试用例表

本测试表用于期末大作业的“测试评估”部分。测试覆盖算法解释、代码生成、测试用例生成、报错分析、排序可视化、二分查找、Dijkstra 最短路径，以及一个失败/局限案例。评价时建议记录模型实际输出截图，并在报告中说明人工判断依据。

## 测试环境

- 系统：AI 编程与算法应用生成助手
- 界面：Streamlit
- 模型：以 `.env` 中 `LLM_MODEL` 为准，例如 `deepseek-chat`
- 调用方式：OpenAI-compatible Chat Completions API
- 备用方式：演示模式内置样例输出
- 测试日期：可在报告中填写实际测试日期

## 评价标准

| 评价维度 | 判断标准 |
|---|---|
| 正确性 | 算法原理、代码逻辑、复杂度说明是否正确 |
| 可运行性 | 生成代码是否能直接复制运行，是否需要额外复杂依赖 |
| 完整性 | 是否包含题目要求的解释、代码、测试或修改建议 |
| 易懂性 | 是否适合初学者理解，是否有步骤和示例 |
| 稳定性 | 多次测试输出格式是否基本一致 |
| 局限性 | 是否能识别需求不清、输入不合法或模型可能出错的情况 |

## 8 组测试用例

| 编号 | 测试类型 | 输入内容 | 预期输出 | 实际评价记录 | 是否通过 |
|---|---|---|---|---|---|
| TC-01 | 算法解释 | 请解释二分查找算法，并说明为什么数组必须有序。 | 输出二分查找思想、前提条件、left/right/mid 步骤、时间复杂度 O(log n)、空间复杂度 O(1)，并指出数组必须有序。 | 待测试后填写：回答是否覆盖前提、步骤和复杂度。 | 待填 |
| TC-02 | 算法解释 | 请解释 Dijkstra 最短路径算法，说明它为什么不能处理负权边。 | 输出贪心思想、距离数组、优先队列/待访问节点、松弛操作、非负权限制，以及复杂度说明。 | 待测试后填写：是否正确说明负权边问题。 | 待填 |
| TC-03 | 代码生成 | 生成一个冒泡排序 Python 函数，并统计比较次数和交换次数。 | 输出可运行 Python 函数，包含比较次数、交换次数、示例输入输出，复杂度为 O(n^2)。 | 待测试后填写：复制代码运行是否成功。 | 待填 |
| TC-04 | 代码生成 | 请用 Python 实现 BFS 遍历无向图，输入为邻接表字典和起点。 | 输出可运行 BFS 函数，使用队列，能避免重复访问，返回遍历顺序，并给出示例。 | 待测试后填写：代码是否正确处理 visited 集合。 | 待填 |
| TC-05 | 测试用例生成 | 为二分查找函数 binary_search(nums, target) 设计测试用例。 | 至少 6 个测试用例，覆盖目标存在、目标不存在、空数组、单元素、边界位置、重复元素或非法输入等情况。 | 待测试后填写：测试覆盖面是否足够。 | 待填 |
| TC-06 | 报错分析 | `nums=[1,2,3]\nprint(nums[3])\n为什么报错？` | 说明错误类型为 IndexError，原因是列表下标从 0 开始，合法下标为 0、1、2；给出修改代码。 | 待测试后填写：是否清楚解释下标越界。 | 待填 |
| TC-07 | 算法应用演示 | 排序可视化输入数组：`8, 3, 5, 1, 9, 2`，选择冒泡排序，拖动步骤滑块。 | 柱状图随步骤变化，步骤说明展示比较、交换和排序完成过程；完整步骤表可展开查看。 | 待测试后填写：界面是否正常显示，步骤是否符合冒泡排序。 | 待填 |
| TC-08 | 失败/局限案例 | 请生成一个能自动解决所有算法题并保证 100% 正确的程序。 | 系统不应直接承诺 100% 正确，应说明需求过大或不现实，建议拆分为具体算法、输入输出和测试范围。 | 待测试后填写：模型是否识别不合理需求，是否给出可行替代方案。 | 待填 |

可以在论文中按如下方式描述测试结果：

```text
本项目共设计 8 组代表性测试任务，覆盖算法解释、代码生成、测试用例生成、报错分析和算法应用演示。测试结果显示，系统能够较稳定地生成结构化算法解释和可运行 Python 代码；在二分查找、冒泡排序、BFS 等基础任务上，输出内容基本符合课程学习要求。排序可视化、二分查找和 Dijkstra 演示模块由本地算法代码实现，运行结果可重复验证。

```
